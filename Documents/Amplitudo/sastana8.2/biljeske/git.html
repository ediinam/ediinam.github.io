Introduction
  1. Repository is usually used to ogranize a single project.Repositories can contain folders and iles,images,videos ect.Recommended use od README with information about project.
  2. Branching is a way to work on different versions of a repository at one time. By default we have master wich is considered to be the definitive branch. We use other branches to make edits before committing them to master. When we create a branch off the master, we make a copy of the master branch
  3. Saved branches are called commits, each commit has an associated commit message, a description of changes made.
  4. When you open a pull request, you're proposing changes and requesting that someone review and pull your contribution and megre them into their branch. Pull requests show diffs of differences from both branches which are shown in green and red.
  5. After pull request comes merge where you merge edits from side branch to master branch.
    Git commands
  • git init - creates a new repository
    • git clone path/to/repository - create a working copy of a local repository
    • git clone username@host:/path/to/repository - create a copy from remove server
    • git add filename - propose changes and than git commit -m "Commit message" - now the file is commited to head but not in remote repository
    • git add * - propose changes and than git commit -m "Commit message" - now the file is commited to head but not in remote repository
    • git push origin master - promjene saljemo sa grane u kojoj smo na master granu na lokalni repo
    • git push -u origin master - saljemo master granu na remote
    • git push origin branchname - saljemo svoju granu na remote repo
    • git push --all origin - sve grane pushamo na remote
    • git push origin :branchname - brise granu sa remote
    • git branch ime_grane - pravimo branch
    • git branch - listamo sve grane
    • git branch -d branchname - brise granu
  • git checkout ime_grane - prebacujemo se na granu
  • git merge ime_grane - prebacimo izmijenjene fajlove sa ime_grane na onu u kojoj se nalazimo
  • git pull origin remote - da bi sa remote prebacili na local git pull - preuzima sve promjene sa remote
    • git remote add origin server - ako nismo povezali projekat, povezemo ga sa remote repo kako bi pushali
    • git remote -v - prikazuje sve remote repo
  • git diff - prikazuje konflikte
  • git status - prikazuje izmijenjene fajlove koji nisu commitovani
  • git log - prikazuje sve izmjene
    Koraci za pravljenje projekta
  1. Napravimo folder u kojem zelimo da bude projekat
  2. U tom folderu otvorimo git terminal
  3. git init
  4. U folderu napravimo fajl
    • git add .
    • git commit -m "Commit message"
  5. Koristimo git status da bi vidjeli koje su promjene napravljene
  6. Ako smo nesto izmijenili,
    • git add "ime fajla"
    • git commit -m "poruka"
  7. Kad kreiramo granu, prebacimo se u nju, dodajemo joj fajlove sa git add i git commit
  8. Pravimo izmjene na fajlovima na sporednoj grani
  9. Prebacimo se na glavnu granu i tu kucamo git merge ime_sporedne_grane kako bi spojili promjene
  10. Da bi prebacili projekat na remote repository odemo na GitHub profil i samo kreiramo repository
  11. Uzmemo link repository-ja
  12. git remote add origin url postavljamo origin remote
  13. Da bi pushali kod na remote radimo git push -u origin master sa glavne grane na remote repo
